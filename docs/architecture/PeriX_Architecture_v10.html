安全与治理（全局贯穿）

L1 接入与连接层 用户触点 · 协议 · 消息平台 · Gateway

▶

所有用户和外部系统的统一入口。原生客户端经由 L2 感知与执行层到达核心引擎；消息平台、API/SDK 及 Agent 协议直达 L3 核心引擎层。所有请求经过 Gateway 统一治理后进入后端服务。

原生客户端

  • iOS App (Swift / SwiftUI)
  • Android App (Kotlin / Compose)
  • Desktop Agent (Tauri)
  • Web Dashboard (React / Next.js)
  • CLI 工具 (synctl)
  • 兼容前端 (Open WebUI / LobeChat)

消息平台适配

  • 微信 / 企业微信
  • 钉钉 / 飞书
  • Telegram / Discord / Slack
  • WhatsApp / Signal / LINE
  • iMessage (BlueBubbles)
  • 统一会话路由 & 上下文管理

Agent 互操作协议

  • MCP (Model Context Protocol) — 工具/资源发现与调用
  • A2A (Agent-to-Agent) — 跨 Agent 通信
  • ACP (Agent Communication Protocol) — 编辑器/IDE 集成
  • MCP Server Cards / A2A Agent Cards (发现机制)

传输协议 & 开发者 SDK

  • REST API (OpenAI 兼容格式)
  • gRPC (强类型, 高吞吐)
  • WebSocket (双向实时交互)
  • SSE (流式推送, MCP 远程传输)
  • STDIO (MCP 本地传输)
  • Webhook (外部事件回调)
  • SDK (Python / TypeScript / Go)
  • OpenAPI / Protobuf 接口规范

Gateway（物理部署在云端）

  • 请求路由 & 负载均衡
  • 协议转换 (REST ↔ gRPC ↔ WS ↔ SSE)
  • SSL/TLS 终止
  • X-Session-Id 持久会话管理
  • 服务发现 & 健康检查
  • 请求/响应转换 & 聚合
  • 治理策略执行点（策略定义在 L0）

L2 感知与执行层 情境感知 · 本地工具 · 交互渲染 · 隐私管理（物理部署在用户终端）

▶

运行在用户终端设备上的感知和执行运行时（仅原生客户端路径经过本层；消息平台与 API 请求直达 L3）。负责采集情境信号、执行核心引擎下发的任务指令、渲染交互界面、管理本地隐私数据。通过 MCP 协议与 L3 形成"感知上报 → 决策 → 执行 → 反馈"闭环。

情境感知引擎

  • GPS / 地理围栏 / 位置变化
  • 日历 / 日程 / 提醒事项监听
  • 系统传感器 (运动, 光感, 电量)
  • 屏幕状态 / App 使用行为
  • 消息流监听 (用户授权)
  • 语义快照 (Semantic Snapshot)NEW
  • 情境信号聚合 & 上报

本地工具执行引擎

  • 设备工具注册表 (文件/日历/通讯录/相册/摄像头)
  • 设备权限 & 授权管控
  • MCP Server (暴露本地工具给 L3 调用)
  • App 间调用 (Deep Link / Intent / URL Scheme)
  • 剪贴板 / 通知 / 系统设置
  • 离线工具执行 & 结果缓存

客户端运行时

  • MCP 协议引擎 (与 L3 的工具桥梁)
  • 轻量 Agent 执行器 (L3 指令执行)
  • Lane Queue 三模式 (steer/followup/collect)NEW
  • 离线任务队列 & 重试
  • 双向通信 (WS / SSE)

交互与渲染

  • Feed 卡片流 UI
  • 主动推送消息展示
  • 一键执行交互 (确认/回复/预约)
  • 语音 / 文本 / 多模态输入
  • 跨端一致体验

本地数据管理

架构设计原则

1. 感知-决策-执行闭环：L2 感知 → L3 决策 → L2 执行 → 反馈学习，形成完整数据闭环。消息平台/API 路径跳过 L2 直达 L3。
2. 逻辑分层 ≠ 物理部署：层按功能职责划分，非物理位置。L1 Gateway 物理在云端集群，L2 物理在用户终端，逻辑上各自独立。
3. 分级记忆 + 用户建模：工作记忆 → 短期记忆 → 长期记忆（毕业机制），独立用户画像系统持续深化理解。
4. 双进程分离：API Server 负责管理面，Actor Pool 负责执行面，解耦扩展。
5. 安全纵深：从网关鉴权 → 数据隔离 → AI 护栏 → 执行沙箱 → 审计日志，层层设防。
6. 自演进四维度：Prompt 优化 + 架构演化 + 知识积累 + Skill 自动生成，Agent 持续自我进化。
7. 事件驱动解耦：消息队列连接执行层、演进引擎、推送引擎、记忆沉淀管道，异步处理。

v8 变更日志（vs v7）

v7 全部变更保留（层名逻辑化、记忆重构、用户建模、审批门控、Checkpoint、智能静默、Skill 自生成等）。

v8 新增变更：
L0 治理层重构（5域 → 7域）：
  • "AI 安全" → Agent 安全（精准定位 Agent 行为层面安全）
  • 新增 Agent 治理：Agent 生命周期、发布灰度、行为边界、资源配额、版本回滚、演化审批
  • 新增 服务治理：限流配额、DDoS、灰度流量、API 版本、熔断降级、链路追踪
  • "合规与审计"中 Agent 演化审批迁入 Agent 治理
L1 Gateway 瘦身：治理类内容迁出至 L0，Gateway 仅保留纯功能（路由、协议转换、SSL、会话、服务发现），标注"治理策略执行点（策略定义在 L0）"。
